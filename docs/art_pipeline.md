# Fabled Mercenaries — 아트 파이프라인 방침

> 1인 개발 제약 분석 후 정리한 아트 생산 전략. 작성: 2026-06-26.
> 관련: `STATUS.md`, `combat_class_design.md`.

---

## 0. 전제 (제약 재정의)
- **1인 프로그래머. 코드/개발 시간은 충분**(업무 중 여유 시간 활용). → 프로그래밍 스코프는 병목 아님.
- **진짜 병목 = 아트, 특히 3D 모델링.** 일러스트는 AI로, 모델링은 약간의 학습 필요 — 이쪽이 최대 리스크.

## 1. 설계가 이미 모델링 부담을 낮춘다
- **SD(3~4등신) + 카툰 + 쿼터뷰** = 모델링 난이도 최하 구간. 단순 형태·적은 디테일·관대한 토폴로지.
- **쿼터뷰**(카메라 멀음) → 얼굴/디테일 거의 안 보여 매우 용서됨.
- **게이지·명령 기반 전투**(트위치 액션 아님) → 화려한 모션 의존도 낮음. 기능적 애니만으로도 성립.

## 2. "모델러 안 되고도" 3D 채우는 전략
- **Synty POLYGON 시리즈** ← 솔로 개발 사실상 표준. 스타일 통일된 로우폴리 캐릭터/환경 모듈 팩, 쿼터뷰에 적합. 베이스 구매 후 커스터마이즈.
- **Mixamo**(무료, Adobe) — 휴머노이드 **자동 리깅 + 애니메이션 라이브러리**. "클래스·스킬마다 모션" 문제를 크게 해소.
- **VRoid Studio**(무료) — 애니풍 캐릭터 생성기, 리깅된 채 익스포트.
- **AI 3D 생성**(Meshy / Tripo / Rodin 류, 채택 시 당해 품질 확인) — 소품·무기·베이스 메시에 유효. 캐릭터는 토폴로지/리깅 정리 필요하나 "0에서"보다 훨씬 빠름.
- **모듈러 / 킷배싱** — 클래스마다 통짜 모델 ✗. **베이스 용병 1개 + 무기·방어구 교체**로 다클래스 표현. 전직/강림 시스템과 궁합 좋음.

## 3. 실제 학습할 최소 모델링 (우선순위)
풀 캐릭터 모델링은 목표 아님. "고치고 조립할 정도"면 충분.
1. **Blender 기초** — 임포트/익스포트, 간단 수정, 리토포 감각.
2. **리깅/스키닝 기본** — 또는 Mixamo 자동 리깅에 의존.
3. **UE 머티리얼/툰 셰이더** — 사실상 코드/수학 = **프로그래머 강점**. 셰이딩으로 스타일을 만들어 모델 단순함을 가린다. 여기에 기대라.
- 직접 모델링은 **소품·무기**(쉬움)부터, 캐릭터는 베이스 에셋 활용.

## 4. 주의점 (폴리시)
- **스타일/소스 통일이 폴리시의 핵심.** AI 메시 + 마켓 에셋 + 다른 출처를 섞으면 통일성이 깨져 싸구려로 보임. → **한 스타일/소스 정해 끝까지 통일**(Synty 인기 이유 = 일관성).
- **애니메이션 손맛**: Mixamo 기본만으론 평범. **핵심 공격/스킬 모션**은 손 또는 AI 애님 툴로 다듬어야 살아남.
- 강림/전직 변형 = 캐릭터 변종 多 → **모듈러 + 스타일 통일**로 양 부담 완화.

## 5. 권장 파이프라인 (요약)
- **캐릭터**: Synty/VRoid/AI 베이스 → Mixamo 리깅·애님 → 툰 머티리얼 → 무기·방어구 모듈 교체로 클래스 분화.
- **환경**: Synty POLYGON 모듈 키트(드래그드롭, 통일성).
- **테크아트/셰이더**: 프로그래머 강점 영역으로 적극 활용(아웃라인·툰·이펙트).
- 핵심 원칙: **모델링을 "마스터"가 아니라 "베이스 에셋 + 자동리깅 + 셰이더 + 모듈러 조립"으로 우회.**
